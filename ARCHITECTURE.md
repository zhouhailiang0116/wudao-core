> 维护者：八戒

## 顶层架构

```
业（决策）
    ↓ 咒语
道（策划/记忆中枢）
    ↓ 理解
七公理（数学法则）
    ↓ 翻译
通层 bridge（axiom ↔ Python）
    ↓ 执行
GPU着色器 / FPGA / ASIC
```

---

---

## 数据流

```
用户输入（咒语/指令）
    ↓
道理解意图 → 选择对应axiom
    ↓
bridge翻译 → Python函数调用
    ↓
GPU着色器渲染 / NumPy计算
    ↓
结果输出 → 窗口/文件/内存
```

---

## 核心文件

| 文件 | 职责 |
|------|------|
| `axiom_loader.py` | 自动扫描并加载所有axiom模块 |
| `AgentCore.py` | Agent核心框架（状态机+记忆+错误处理） |
| `bridge/axiOM.md` | axiom到Python的翻译映射表 |
| `demo_closed_loop.py` | 端到端回归测试 |

---

## 设计意图

1. **数学先行**：每个axiom先有数学表达，再有代码实现
2. **层层蒸馏**：数学 → Python → NumPy → GPU → FPGA → ASIC
3. **桥接解耦**：bridge层隔离数学层和实现层，方便换底层
4. **文档即记忆**：代码改动必须同步更新本文档
